Contributing to Specimen RO‑9
Thanks for your interest in contributing to the Specimen RO‑9 project.
This repository is intended to be a clean, transparent reference for DayZ modding workflows, and contributions that improve clarity, structure, or functionality are welcome.

How to Contribute
1. Keep the Code Clean
- 	Follow the existing folder structure and naming conventions
- 	Keep scripts and configs readable and documented
- 	Avoid committing build artifacts (, , , etc.)
2. Respect Asset Restrictions
Do not submit models, textures, or audio assets unless:
- 	you created them yourself, and
- 	you are willing to license them appropriately.
This project cannot accept copyrighted assets from external sources.
3. Submit Pull Requests
If you want to propose a change:
1. 	Fork the repository
2. 	Create a feature branch
3. 	Make your changes
4. 	Submit a pull request with a clear description of what you’ve done
Small, focused PRs are easier to review and merge.
4. Report Issues
If you find a bug or have a suggestion:
• 	Open an issue
• 	Provide clear steps, logs, or screenshots if relevant
• 	Keep discussions constructive and technical

Areas Where Help Is Especially Welcome
- 	Animation setup
- 	Skeleton authoring
- 	AI behavior scripting
- 	Encounter/event design
- 	Sound design
- 	Documentation improvements

Code of Conduct
Be respectful, constructive, and collaborative.
This project is meant to help the DayZ modding community grow, and contributions should reflect that spirit
